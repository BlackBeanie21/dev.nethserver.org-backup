CoovaChilli hangs when configured with external dns and network is down

Added by Giovanni Bezicheri over 5 years ago. Updated over 5 years ago.

Status:CLOSEDStart date:
Priority:NormalDue date:
Assignee:-% Done:

100%

Category:coova-chilli
Target version:v6.7
Security class: Resolution:
Affected version:v6.7 NEEDINFO:No

Description

CoovaChilli hangs on starting when internet connection is not ready.

Steps to reproduce:
  1. configure your machine with a red interface, a green interface and another interface (ethX) connected to a guest network
  2. install coova-chilli, edit /etc/chilli/defaults and set HS_LANIF to ethX
  3. restart coova-chilli
  4. disconnect the red interface from the internet connection
  5. restart your machine and check that chilli hangs
