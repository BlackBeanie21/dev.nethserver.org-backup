Mail-common: timeout after END-OF-MESSAGE from localhost

Description

Our maillog is filled with timeout after END-OF-MESSAGE from localhost messages from postfix/smtpd.

amavisd has a SMTP connection caching mechanism that can be disabled, to avoid those messages on low-traffic sites.

http://permalink.gmane.org/gmane.mail.postfix.user/186129

Two amavisd.conf parameters control the smtp connection cache: 

  $smtp_connection_cache_on_demand = 1;
  $smtp_connection_cache_enable = 1;

From amavisd 2.6.0 release notes:

A global setting $smtp_connection_cache_on_demand controls whether amavisd
is allowed to dynamically change the $smtp_connection_cache_enable setting
according to its estimate of the message frequency.

There should be no appreciable performance loss if we disable the connection cache, because amavisd contacts postfix without TLS on the loopback interface.
