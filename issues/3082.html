Unexpected ethernet interface rename

Added by Giacomo Sanchietti over 6 years ago. Updated over 6 years ago.

Status:CLOSEDStart date:
Priority:NormalDue date:
Assignee:-% Done:

100%

Category:nethserver-base
Target version:v6.6
Security class: Resolution:
Affected version:v6.6-rc1 NEEDINFO:No

Description

Steps to reproduce
  • On a system with only one interface (eth0) add a new network interface
  • After reboot, the new interface is named eth2 and not eth1
Expected result
  • The new interface should be named eth1

When a new interface is added, udev triggers an event with a custom scripts.
If the script fails (eg. the filesystem is not mounted), the event is triggered again and multiple lines for the same mac address appear inside /etc/udev/rules.d/70-persistent-net.rules file.
