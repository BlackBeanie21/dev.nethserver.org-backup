Create User tab not reset

Added by Davide Principi over 7 years ago. Updated about 6 years ago.

Status:CLOSEDStart date:
Priority:NormalDue date:

100%

Target version:v6.6

Affected version:v6.5-final NEEDINFO:No

Description

Subsequent creations of new user accounts "remember" the last selected form tab.

To reproduce the problem:
- start creating a new user, filling required fields
- move to "Services" tab
- Save
- click "Create new" again

The "Services" tab is already selected.

The expected behaviour is to have the "General" tab (the first) selected.
