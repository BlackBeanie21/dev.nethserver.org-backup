Bug #2205

/etc/sysconfig/system-config-firewall not expanded by lokkit-save event

Added by Davide Principi almost 8 years ago. Updated almost 8 years ago.

Status:CLOSEDStart date:
Priority:NormalDue date:
Assignee:-% Done:

100%

Category:nethserver-base
Target version:v6.4-beta2
Security class: Resolution:
Affected version:v6.4-beta1 NEEDINFO:No

Description

The lokkit-save event does not expand /etc/sysconfig/system-config-firewall template. I've noticed it after installing nethserver-httpd: the port 80 is never opened.

As a workaround I typed:

   # expand-template /etc/sysconfig/system-config-firewall
   # signal-event firewall-adjust
