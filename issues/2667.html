Bug #2667

Backup data: can't install package if backup is in progress

Added by Giacomo Sanchietti over 7 years ago. Updated over 7 years ago.

Status:CLOSEDStart date:
Priority:NormalDue date:
Assignee:-% Done:

100%

Category:nethserver-backup-data
Target version:v6.5-rc1
Security class: Resolution:
Affected version:v6.5-beta3 NEEDINFO:No

Description

If a backup is in progress it's not possible to update nethserver-backup-data rpm due to a cpio error.
The error occurs when /mnt/backup is mounted because it's part of nethserver-backup-data package.

Solution
Remove /mnt/backup directory from nethserver-backup-data rpm and create the directory inside the backup script.
