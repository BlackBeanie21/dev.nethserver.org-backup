Bug #2094

RemoteAccess/HttpdAdmin UI module does not expand httpd-admin configuration

Added by Davide Principi about 8 years ago. Updated almost 8 years ago.

Status:CLOSEDStart date:08/22/2013
Priority:NormalDue date:08/22/2013
Assignee:-% Done:

100%

Category:nethserver-httpd-admin
Target version:v6.4-beta2
Security class: Resolution:
Affected version:v6.4-beta1 NEEDINFO:No

Description

Package version:
nethserver-httpd-admin-1.0.4-1.ns6.noarch

Symptom:
after saving RemoteAccess UI form, changes are not applied to httpd-admin

  # find /etc/e-smith/events/remoteaccess-update/
/etc/e-smith/events/remoteaccess-update/
/etc/e-smith/events/remoteaccess-update/services2adjust
/etc/e-smith/events/remoteaccess-update/services2adjust/httpd-admin
/etc/e-smith/events/remoteaccess-update/services2adjust/sshd
/etc/e-smith/events/remoteaccess-update/S20firewall-adjust
/etc/e-smith/events/remoteaccess-update/templates2expand
/etc/e-smith/events/remoteaccess-update/templates2expand/etc
/etc/e-smith/events/remoteaccess-update/templates2expand/etc/ssh
/etc/e-smith/events/remoteaccess-update/templates2expand/etc/ssh/sshd_config
Workaround:
  • Expand /etc/httpd/admin-conf/httpd.conf template
  • Reload httpd-admin service
