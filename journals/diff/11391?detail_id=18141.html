Bug #2780

After "migration migration from NethService/SME Server":http://docs.nethserver.org/en/latest/migration.html Server (http://docs.nethserver.org/en/latest/migration.html) you cannot access samba shares with migrated users:<pre>
imported users:

# smbclient //localhost/miaibay -U nico
Enter nico's password:
session setup failed: NT_STATUS_LOGON_FAILURE
</pre>

Examining imported users properties, they all have no flag over "Shared Folders (Samba)" option "Services" tab.
Even flagging the option the problem remains:<pre>
remains:

# smbclient //localhost/miaibay -U nico
Enter nico's password:
session setup failed: NT_STATUS_LOGON_FAILURE
</pre>


Confirming user's password through "change password" option works around solves the problem.
